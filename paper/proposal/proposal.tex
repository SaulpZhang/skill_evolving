\section{Introduction}
\label{sec:introduction}

\subsection{Background and Motivation}

Large language model (LLM) agents have demonstrated the ability to perform complex tasks across a variety of environments, including embodied task completion, web navigation, and software engineering. A key capability enabling such generality is \emph{skill evolution}: the agent reflects on its execution trajectories, extracts reusable knowledge, and accumulates it in a growing skill library for future episodes.

Prior work has focused primarily on two aspects of this process. The first is \emph{skill quality}---how to extract more informative patterns from trajectories. For instance, SkillRL introduces a reflection mechanism that generates planning patterns from successful episodes and common mistakes from failures. The second is \emph{skill utilization}---how to organize, retrieve, and inject skills into the agent's prompt at inference time. These efforts include structured skill banks with category-specific retrieval, vector-database approaches, and dynamic prompting strategies.

A critical dimension has remained largely unexplored: \textbf{the order in which tasks are selected for practice.} Existing systems either cycle through a fixed task pool uniformly, sample tasks at random, or rely on hand-designed curricula that do not adapt to the agent's evolving competence. In RL, it is well-known that the task selection strategy can significantly affect learning efficiency. So, it is natural to ask: \emph{can we improve the efficiency and performance of skill evolution in LLM agents by adaptively selecting tasks based on the agent's current skill profile?}

\subsection{Contributions}

We present \textbf{SPG-Bandit} (Skill Profile Guided Bandit), a framework that casts task selection as a contextual bandit problem driven by a multidimensional skill profile of the agent.

Our core idea is to model the agent's latent proficiency across \(K\) skill dimensions using Multidimensional Item Response Theory (MIRT), then select tasks that maximize the expected progress toward underdeveloped dimensions.

Specifically, this proposal makes the following contributions:
\begin{enumerate}
    \item \textbf{Problem.} We identify and formalize the task selection problem in LLM agent skill evolution. We show that uniform and random selection---the default in existing systems---is suboptimal because different tasks contribute unequally to skill growth and the agent's skill profile is inherently multidimensional.
    \item \textbf{Method.} We propose SPG-Bandit, a framework that combines MIRT-based latent skill profiling with a gap-weighted bandit algorithm. The core idea is to maintain an online estimate of the agent's proficiency across \(K\) skill dimensions, then select tasks that maximize predicted progress toward underdeveloped dimensions.
    \item \textbf{Framework.} We provide a general framework which can easily combine with existing skill evolution methods. The framework is used to evaluate the effectiveness of SPG-Bandit in improving skill evolution efficiency and performance.
\end{enumerate}

\section{Problem Formulation}
\label{sec:problem}

\subsection{Notation and Setup}

Consider an LLM agent interacting with a fixed pool of tasks \(\mathcal{T} = \{1, 2, \dots, M\}\). At each discrete step \(t = 1, 2, \dots\), the agent selects a task \(\tau_t \in \mathcal{T}\), executes it, and receives a binary outcome \(y_t \in \{0, 1\}\) indicating task success or failure.

The agent possesses a \emph{latent skill profile} \(\mathbf{s}_t \in [0,1]^K\) representing its proficiency across \(K\) distinct skill dimensions. The profile is not directly observable; it must be inferred from observed outcomes. We denote the \emph{skill progress} between consecutive steps as \(\boldsymbol{\delta}_t = \mathbf{s}_{t+1} - \mathbf{s}_t\), which measures how much the agent's latent proficiency changes after executing task \(\tau_t\) and updating its skills via reflection.

Each task \(\tau\) is associated with a \emph{task embedding} \(\mathbf{e}_\tau \in \mathbb{R}^{Z^+}\), typically derived from a sentence encoder applied to the task's natural-language goal description.

\subsection{Problem Definition}

We formalize adaptive task selection as a \emph{contextual bandit} problem. At each step \(t\):

\begin{enumerate}
    \item The agent observes its current context: the current skill profile \(\mathbf{s}_t\) and the embedding \(\mathbf{e}_\tau\) for each task \(\tau \in \mathcal{T}\).
    \item The agent selects an arm \(\tau_t \in \mathcal{T}\).
    \item The agent executes task \(\tau_t\), observes binary outcome \(y_t\), and updates its skills via reflection.
    \item The latent skill profile updates to \(\mathbf{s}_{t+1}\).
\end{enumerate}

To prioritize underdeveloped skill dimensions, we define a \emph{gap weight} vector:

\begin{equation}
    \mathbf{g}_t = \operatorname{softmax}\!\left( \frac{\mathbf{1} - \mathbf{s}_t}{\tau_g} \right),
    \label{eq:gap-def}
\end{equation}

where \(\tau_g > 0\) is a temperature parameter. We define the immediate reward as the gap-weighted post-execution profile value:

\begin{equation}
    r_t = \mathbf{g}_t^\top \mathbf{s}_{t+1}.
    \label{eq:reward}
\end{equation}

The goal is to maximize the expected cumulative reward:

\begin{equation}
    \max_{\pi} \; \mathbb{E}\left[ \sum_{t=1}^{H} r_t \;\middle|\; \tau_t \sim \pi(\cdot \mid \mathbf{s}_t, \{\mathbf{e}_\tau\}) \right],
    \label{eq:objective}
\end{equation}

where \(\pi\) denotes the task selection policy. This formulation naturally prioritizes underdeveloped dimensions via the gap weights \(\mathbf{g}_t\), without requiring an auxiliary objective.

Defining the per-step skill progress as \(\boldsymbol{\delta}_t = \mathbf{s}_{t+1} - \mathbf{s}_t\), we have \(r_t = \mathbf{g}_t^\top (\mathbf{s}_t + \boldsymbol{\delta}_t) = \mathbf{g}_t^\top \mathbf{s}_t + \mathbf{g}_t^\top \boldsymbol{\delta}_t\). The term \(\mathbf{g}_t^\top \mathbf{s}_t\) is constant with respect to the choice of \(\tau\) under a given \(\pi\). Hence, maximizing \(r_t\) is equivalent to maximizing the gap-weighted skill progress at each step. We therefore adopt the following equivalent reward:

\begin{equation}
    r_t = \mathbf{g}_t^\top \boldsymbol{\delta}_t.
    \label{eq:reward-delta}
\end{equation}

We learn a surrogate model \(\hat{\boldsymbol{\delta}}(\tau, \mathbf{s}_t)\) and use \(\mathbf{g}_t^\top \hat{\boldsymbol{\delta}}_\tau\) as the reward for bandit decision-making.


\section{Method}
\label{sec:method}

\subsection{Overview}

SPG-Bandit operates in two phases. In the \textbf{warmup phase}, the agent selects tasks uniformly to collect an initial set of interaction outcomes. After warmup, we run Multidimensional Item Response Theory (MIRT) to estimate the agent's latent skill profile for each warmup step, producing a sequence of skill deltas \(\{\boldsymbol{\delta}_t\}\) and item parameters for all tasks. We then train an MLP that predicts these deltas from task embeddings and the current profile.
    
In the \textbf{bandit phase}, at each step we compute a skill-progress gap vector \(\mathbf{g}_t\) from the current profile, use the Neural Linear UCB to predict each task's expected delta, and select the task with the highest gap-weighted UCB score. After execution, we update the profile via a Bayesian online MIRT and update the bandit's ridge regression parameters.

\subsection{Key Components}

\paragraph{Multidimensional IRT (MIRT).}
We model each task \(\tau\) with a \(K\)-dimensional discrimination vector \(\mathbf{a}_\tau \in \mathbb{R}^K\) and a difficulty scalar \(d_\tau \in \mathbb{R}\). Given the agent's current skill profile \(\mathbf{s} \in [0,1]^K\), the probability of success on task \(\tau\) is:

\begin{equation}
    p_\tau(\mathbf{s}) = \sigma\!\left( \mathbf{a}_\tau^\top \mathbf{s} - d_\tau \right),
    \label{eq:mirt}
\end{equation}

where \(\sigma(x) = 1/(1 + e^{-x})\). We constrain \(\mathbf{a}_\tau \geq 0\) and \(d_\tau \geq 0\) element-wise. The non-negativity of \(\mathbf{a}_\tau\) guarantees that increasing any dimension of \(\mathbf{s}\) never decreases the predicted success probability---skill acquisition and task performance are monotonically aligned. Together with \(d_\tau \geq 0\), the model encodes the natural prior that higher proficiency corresponds to higher success, establishing \(\mathbf{s}\) as a directionally reliable indicator of agent capability.

After collecting \(N\) warmup outcomes \(\{( \tau_t, y_t )\}_{t=1}^N\), we estimate the skill profiles and item parameters via alternating optimization:

\textit{E-step.} Fix the item parameters \(\{(\mathbf{a}_\tau, d_\tau)\}\). For each step \(t\), compute the most likely profile \(\mathbf{s}_t\) by maximizing the log-posterior \(\log P(y_t \mid \mathbf{s}_t, \mathbf{a}_{\tau_t}, d_{\tau_t}) + \log P(\mathbf{s}_t)\), where the prior \(P(\mathbf{s}_t)\) is a Gaussian centered at \(0.5\).

\textit{M-step.} Fix the profiles \(\{\mathbf{s}_t\}\). For each task \(\tau\), maximize the log-likelihood of all observations of that task with respect to \((\mathbf{a}_\tau, d_\tau)\) under L-BFGS-B, enforcing \(\mathbf{a}_\tau \geq 0, d_\tau \geq 0\) and \(\ell_2\) regularization on \(\mathbf{a}_\tau\). The two constraints together guarantee \(\mathbf{s} \uparrow \Rightarrow p_\tau(\mathbf{s}) \uparrow\) (higher proficiency monotonically implies higher success probability).

Alternate E- and M-steps until the log-likelihood stabilizes.

\textit{Online update.} In the bandit phase, each new observation \((y_t, \tau_t)\) refines the profile online without re-running the full alternating estimation. Given the current profile \(\mathbf{s}_t\) and task parameters \((\mathbf{a}_{\tau_t}, d_{\tau_t})\), the posterior profile is:

\begin{equation}
    \mathbf{s}_{t+1} = \arg\max_{\mathbf{s} \in [0,1]^K} \; \log P(y_t \mid \mathbf{s}, \mathbf{a}_{\tau_t}, d_{\tau_t}) + \log P(\mathbf{s} \mid \mathbf{s}_t),
\end{equation}



\paragraph{Delta prediction via MLP.}
To anticipate how much a candidate task will improve the agent, we train a two-layer MLP \(\Phi\) that maps the concatenation of the task embedding \(\mathbf{e}_\tau\) and current profile \(\mathbf{s}\) to a feature vector \(\boldsymbol{\phi} \in \mathbb{R}^{Z^+}\):

\begin{align}
    \mathbf{h} &= \max(\mathbf{W}_1 [\mathbf{e}_\tau; \mathbf{s}] + \mathbf{b}_1, 0), \\
    \boldsymbol{\phi} &= \mathbf{W}_2 \mathbf{h} + \mathbf{b}_2.
\end{align}

A linear head \(\mathbf{W_3}\) predicts the expected delta: \(\hat{\boldsymbol{\delta}} = \mathbf{W_3}^\top \boldsymbol{\phi}\). Training minimizes MSE over the warmup dataset \(\{([\mathbf{e}_{\tau_t}; \mathbf{s}_t^{\text{(bef)}}], \boldsymbol{\delta}_t)\}_{t=1}^N\).

\paragraph{Gap-weighted UCB selection.}
The MLP produces a fixed feature representation \(\boldsymbol{\phi}\) for each task. On top of this, we learn a linear mapping from \(\boldsymbol{\phi}\) to the predicted delta \(\hat{\boldsymbol{\delta}}\) via ridge regression. The weight matrix \(\mathbf{W}\) is the ridge regression estimator:

\begin{equation}
    \mathbf{W} = \mathbf{A}^{-1} \mathbf{B},
    \quad
    \mathbf{A} = \lambda \mathbf{I} + \sum_{i=1}^t \boldsymbol{\phi}_i \boldsymbol{\phi}_i^\top,
    \quad
    \mathbf{B} = \sum_{i=1}^t \boldsymbol{\phi}_i \boldsymbol{\delta}_i^\top,
\end{equation}

giving the prediction \(\hat{\boldsymbol{\delta}}_\tau = \mathbf{W}^\top \boldsymbol{\phi}_\tau\). Here \(\mathbf{A}\) also captures uncertainty: \(\boldsymbol{\phi}_\tau^\top \mathbf{A}^{-1} \boldsymbol{\phi}_\tau\) measures the epistemic uncertainty of the prediction. To balance exploitation and exploration, we define the UCB bandit score:

\begin{equation}
    \text{score}(\tau) = \mathbf{g}_t^\top \mathbf{W}^\top \boldsymbol{\phi}_\tau + \alpha \sqrt{\boldsymbol{\phi}_\tau^\top \mathbf{A}^{-1} \boldsymbol{\phi}_\tau},
    \label{eq:score}
\end{equation}

where the first term exploits the predicted gap-weighted delta and the second provides UCB-style exploration weighted by \(\alpha > 0\).

\paragraph{Ridge regression update.}
After observing the realized delta \(\boldsymbol{\delta}\), we update the bandit's covariance and response:

\begin{align}
    \mathbf{A} &\mathrel{+}= \boldsymbol{\phi} \boldsymbol{\phi}^\top, \\
    \mathbf{B} &\mathrel{+}= \boldsymbol{\phi} \boldsymbol{\delta}^\top, \\
    \mathbf{W} &= \mathbf{A}^{-1} \mathbf{B}.
\end{align}

This ensures that the delta prediction adapts as more data is collected during the bandit phase.

\subsection{Algorithm}

Algorithm~\ref{alg:spg} summarizes the complete procedure.

\begin{algorithm}[t]
\caption{SPG-Bandit}
\label{alg:spg}
\begin{algorithmic}
\STATE {\bf Input:} Task pool \(\mathcal{T}\), warmup steps \(N\), bandit horizon \(H\), MLP feature dim \(d_f\), gap temperature \(\tau_g\), UCB exploration \(\alpha\), regularizer \(\lambda\), skill dimensions \(K\)
\STATE {\bf Phase 1: Warmup}
\FOR{\(t = 1\) to \(N\)}
    \STATE Select \(\tau_t \in \mathcal{T}\) uniformly
    \STATE Execute \(\tau_t\), observe \(y_t \in \{0,1\}\)
    \STATE Store \((\tau_t, y_t)\)
\ENDFOR
\STATE Run sequential MIRT EM on \(\{(\tau_t, y_t)\}\) \(\to\) profiles \(\{\mathbf{s}_t\}\), deltas \(\{\boldsymbol{\delta}_t\}\), item parameters \(\{\mathbf{a}_\tau, d_\tau\}\)
\STATE Train MLP \(\Phi\) on \(\{([\mathbf{e}_{\tau_t}; \mathbf{s}_t^{\text{(bef)}}], \boldsymbol{\delta}_t)\}\)
\STATE Initialize \(\mathbf{A} = \lambda \mathbf{I}_{d_f}\), \(\mathbf{B} = \mathbf{0}_{d_f \times K}\), current profile \(\mathbf{s} = \mathbf{s}_N\)
\STATE {\bf Phase 2: Bandit}
\FOR{\(t = 1\) to \(H\)}
    \STATE Compute gap \(\mathbf{g}_t = \operatorname{softmax}((\mathbf{1} - \mathbf{s}) / \tau_g)\)
    \FOR{each \(\tau \in \mathcal{T}\)}
        \STATE \(\boldsymbol{\phi}_\tau = \Phi([\mathbf{e}_\tau; \mathbf{s}])\), \(\hat{\boldsymbol{\delta}}_\tau = \mathbf{W}^\top \boldsymbol{\phi}_\tau\)
        \STATE \(\text{score}(\tau) = \mathbf{g}_t^\top \hat{\boldsymbol{\delta}}_\tau + \alpha \sqrt{\boldsymbol{\phi}_\tau^\top \mathbf{A}^{-1} \boldsymbol{\phi}_\tau}\)
    \ENDFOR
    \STATE \(\tau_t = \arg\max_\tau \text{score}(\tau)\)
    \STATE Execute \(\tau_t\), observe \(y_t\) and realized \(\boldsymbol{\delta}_t\)
    \STATE \(\mathbf{s} \gets \text{BayesUpdate}(\mathbf{s}, \mathbf{a}_{\tau_t}, d_{\tau_t}, y_t)\)
    \STATE \(\mathbf{A} \mathrel{+}= \boldsymbol{\phi}_{\tau_t} \boldsymbol{\phi}_{\tau_t}^\top\)
    \STATE \(\mathbf{B} \mathrel{+}= \boldsymbol{\phi}_{\tau_t} \boldsymbol{\delta}_t^\top\)
    \STATE \(\mathbf{W} = \mathbf{A}^{-1} \mathbf{B}\)
\ENDFOR
\end{algorithmic}
\end{algorithm}

\section{Theoretical Analysis}
\label{sec:theory}

The agent's skill profile \(\mathbf{s}_t\) evolves over time, making the reward distribution non-stationary. We analyze the dynamic regret of SPG-Bandit with the sliding-window ridge regression described in Section~\ref{sec:method}.

\subsection{Non-stationary Setting}

Recall that the reward is \(r_t = \mathbf{g}_t^\top \mathbf{s}_{t+1}\). The profile evolves from \(\mathbf{s}_t\) to \(\mathbf{s}_{t+1}\) after task execution and reflection. Define the one-step variation \(\Delta_t = \|\mathbf{s}_{t+1} - \mathbf{s}_t\|_2\), and the total variation budget over \(T\) steps:

\begin{equation}
    V_T = \sum_{t=1}^{T-1} \Delta_t.
    \label{eq:variation}
\end{equation}

Since \(\mathbf{s}_t \in [0,1]^K\) is bounded and the agent faces diminishing marginal returns near mastery, the total skill path length grows slower than \(T\).

The gap weight \(\mathbf{g}_t = \operatorname{softmax}((\mathbf{1} - \mathbf{s}_t)/\tau_g)\) inherits this bounded variation: softmax is Lipschitz continuous, so \(\|\mathbf{g}_{t+1} - \mathbf{g}_t\| \leq L \cdot \Delta_t\) for some constant \(L\). Consequently, the reward distribution's drift is governed by the same \(V_T = o(T)\) budget.

\subsection{Assumptions}

\begin{enumerate}
    \item \textbf{(Local linearity.)} Within any sliding window of size \(\tau\), there exists a fixed \(\mathbf{W}^* \in \mathbb{R}^{d_f \times K}\) such that \(\mathbb{E}[\mathbf{s}_{t+1} \mid \boldsymbol{\phi}_{\tau_t}] = \mathbf{W}^{*\top} \boldsymbol{\phi}_{\tau_t}\).
    \item \textbf{(Bounded features.)} \(\|\boldsymbol{\phi}_\tau\|_2 \leq 1\) for all \(\tau \in \mathcal{T}\), achievable by scaling.
    \item \textbf{(Bounded variation.)} \(V_T = \sum \|\mathbf{s}_{t+1} - \mathbf{s}_t\|_2 = o(T)\), as argued above.
\end{enumerate}

Assumption~1 states that the next-step profile is locally linear in the learned feature space. Assumption~2 is standard, and Assumption~3 is the structural condition that keeps the non-stationary setting tractable.

\subsection{Dynamic Regret Decomposition}

The UCB score upper-bounds the true reward with high probability: \(\text{score}(\tau_t) \geq r_t^*\). The instantaneous regret is therefore bounded by the prediction error of the chosen arm:

\[
r_t^* - r_t \leq  \mathbf{g}_t^\top (\mathbf{s}_{t+1} - \hat{\mathbf{s}}_{t+1}).
\]

Inserting the true local linear model \(\mathbf{W}^*\) from Assumption~1 yields two orthogonal error sources:

\[
\hat{\mathbf{s}}_{t+1} - \mathbf{s}_{t+1} = \underbrace{(\mathbf{W}_t - \mathbf{W}^*)^\top \boldsymbol{\phi}_\tau}_{\text{estimation}} \;+\; \underbrace{(\mathbf{W}^{*\top} \boldsymbol{\phi}_\tau - \mathbf{s}_{t+1})}_{\text{model misspecification}}.
\]

\textbf{Estimation error.} The first term arises from learning \(\mathbf{W}_t\) from a finite window of \(\tau\) samples. The self-normalized martingale bound (Abbasi-Yadkori et al., 2011) yields per-step error \(\tilde{O}(1/\sqrt{\tau})\), accumulating to \(\tilde{O}(T / \sqrt{\tau})\) over \(T\) steps.

\textbf{Drift error.} The true profile satisfies Assumption~1 only approximately within the window. The misspecification at step \(t\) is bounded by the cumulative profile drift inside the window:

\[
\|\mathbf{W}^{*\top} \boldsymbol{\phi}_{\tau_t} - \mathbf{s}_{t+1}\| \leq \sum_{k = \max(1, t-\tau)}^{t} \Delta_k.
\]

Over \(T\) steps, each \(\Delta_k\) appears in at most \(\tau\) windows, yielding \(O(\tau \cdot V_T)\).

Assembling both contributions:

\begin{equation}
    R(T) = \sum_{t=1}^T (r_t^* - r_t) \leq
    \underbrace{\tilde{O}\!\left( \frac{T}{\sqrt{\tau}} \right)}_{\text{estimation}} \;+\;
    \underbrace{O(\tau \cdot V_T)}_{\text{drift}}.
    \label{eq:decomposition}
\end{equation}

\subsection{Optimal Window and Regret Bound}

Balancing the two terms by setting \(\frac{T}{\sqrt{\tau}} = \tau \cdot V_T\) yields the optimal window size:

\begin{equation}
    \tau^* = \left( \frac{T}{V_T} \right)^{2/3},
\end{equation}

and the corresponding cumulative regret bound:

\begin{equation}
    R(T) = \tilde{O}\!\left( d_f \cdot T^{2/3} \cdot V_T^{1/3} \right).
    \label{eq:bound}
\end{equation}

Since \(V_T = o(T)\), let \(V_T = O(T^\nu)\) for some \(\nu < 1\). Substituting:

\begin{equation}
    R(T) = \tilde{O}\!\left( T^{2/3} \cdot T^{\nu/3} \right) = \tilde{O}\!\left( T^{(2+\nu)/3} \right) = o(T),
\end{equation}

establishing strictly sublinear regret. For example, \(\nu = 1/2\) (active learning) gives \(\tilde{O}(T^{5/6})\); \(\nu = 0\) (near saturation) gives \(\tilde{O}(T^{2/3})\).
